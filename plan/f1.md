## 3. جميع نقاط API (Endpoints)

### 3.1 المصادقة والحساب
```
POST driver/login                    → تسجيل الدخول
POST driver/loginWithMobile          → دخول برقم الجوال
POST driver/profile                  → الملف الشخصي
POST driver/terms/accept             → قبول الشروط (multipart + توقيع)
GET  driver/terms/get                → جلب نص الشروط
```

### 3.2 إدارة المهام
```
POST driver/tasks                    → قائمة المهام
POST driver/task/start               → بدء المهمة
POST driver/tasks/confirm            → تأكيد استلام المهام
POST driver/task/fromlocation/confirm → تأكيد موقع الاستلام
POST driver/task/tolocation/confirm   → تأكيد موقع التسليم
```

### 3.3 العينات
```
POST samples/add                     → إرسال العينات (بحسب الحرارة)
POST samples/box/add                 → إرسال الصناديق
POST samples/list                    → قائمة العينات
POST samples/bag                     → عينات الكيس
POST samples/bag/type                → عينات حسب النوع
POST samples/container/add           → إضافة للحاوية
POST samples/container/bags/add      → إضافة أكياس للحاوية
POST task/nosamples                  → تسجيل "لا يوجد عينات"
```

### 3.4 حالة المهمة
```
POST task/collect                    → جمع المهمة (مع/بدون توقيع)
POST task/close                      → إغلاق المهمة
POST task/freezer                    → إغلاق الفريزر
POST task/freezer/out                → إخراج من الفريزر
POST tasks/close                     → إغلاق مهام التوصيل
POST tasks/freezer/out               → إغلاق InFreezer→OutFreezer
```

### 3.5 الحاويات والأكياس
```
POST task/containers/bag             → حاويات المهمة
POST container/bags                  → أكياس الحاوية
POST bag/container/remove            → إزالة كيس من حاوية
POST task/bags/get                   → أكياس المهمة
```

### 3.6 الموقع
```
POST task/location/check             → فحص موقع المهمة
POST tasks/location/check            → فحص موقع التوصيل
POST driver/location                 → تحديث موقع السائق
POST driver/client/tasks             → مهام العميل (OutFreezer)
```

### 3.7 التبادل (Swap)
```
POST swap/list/driver                → قائمة طلبات التبادل
POST swap/reject                     → رفض التبادل
POST swap/accept                     → قبول التبادل
POST swap/create                     → إنشاء طلب تبادل
POST swap/receive                    → استلام التبادل
POST swap/list/driver/accept-all     → قبول الكل
POST swap/list/acceptall             → قبول الكل (نسخة قديمة)
```

### 3.8 التحويلات المالية
```
POST money/transfer/list             → قائمة التحويلات
POST money/transfer/otp/from/verifiy → OTP الاستلام
POST money/transfer/otp/to/verifiy   → OTP التسليم
```

### 3.9 أخرى
```
POST driver/car/images               → رفع صور السيارة (multipart)
POST car/release                     → تحرير السيارة
POST driver/notifications            → الإشعارات
POST driver-schedule                 → الجدول
POST driver/schedule/acceptall       → قبول كل الجدول
POST emergency                       → طلب طوارئ
```

## 6. نقاط حرجة يجب الانتباه لها

### 6.1 مسح الباركود (Scandit SDK)
- التطبيق يستخدم Scandit SDK المدفوع + ML Kit كبديل مجاني
- **قرار مطلوب**: الاستمرار مع Scandit Flutter SDK أو الانتقال لـ `mobile_scanner` المجاني

### 6.2 تتبع الموقع في الخلفية
- التطبيق يرسل الموقع كل 30 ثانية عبر Foreground Service
- في Flutter: استخدام `flutter_background_service` + `geolocator`
- يجب اختبار هذا على iOS أيضاً (قيود مختلفة)

### 6.3 إرسال العينات المتسلسل (Chain API Calls)
- العينات تُرسل بـ 3 طلبات متسلسلة حسب الحرارة (Room → Refrigerate → Frozen)
- **لا يوجد rollback** إذا فشل طلب في المنتصف
- **تحسين مقترح**: تجميعها في طلب واحد في Flutter

### 6.4 رفع الصور (Multipart)
- فحص السيارة يرفع 6 صور + توقيع في طلب واحد
- التوقيع يُرفع كملف صورة
- استخدام `dio` مع `FormData` و `MultipartFile`

### 6.5 حالات المهمة (Task Status Flow)
```
NEW → COLLECTED → IN_FREEZER → OUT_FREEZER → CLOSED
                                           → NO_SAMPLES
```

## 9. ملاحظات مهمة

> [!IMPORTANT]
> - التطبيق الحالي Android فقط. Flutter سيتيح دعم **iOS** أيضاً
> - يجب التنسيق مع فريق الـ Backend بخصوص أي تحسينات على الـ API
> - Scandit SDK يحتاج ترخيص منفصل لـ Flutter

> [!WARNING]
> - الـ API الحالي لا يدعم Rollback عند فشل إرسال العينات المتسلسل
> - يُنصح بتوحيد endpoint إرسال العينات في طلب واحد
> - Connection timeout حالياً 60 ثانية - قد تحتاج تعديل

> [!TIP]
