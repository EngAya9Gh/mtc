# f6 — SAMPLES PULL OUT (سحب العينات من الحاوية) — توثيق شامل ومفصّل (نسخة مطوّرة ومبسطة للمطور)
# المصدر الأصلي: TaskTypeFragment ← InFreezerTaskListFragment → InFreezerScanFreezerFragment → InFreezerRemoveBagsFragment
# Base URL: https://gosample.com/api/

---

## نظرة عامة

**SAMPLES PULL OUT** هو الخيار (عادةً) في شاشة اختيار نوع المهمة (`TaskTypeSelectorScreen` / `TaskTypeFragment`).
الهدف منه: **سحب أو استخراج الأكياس** التي تم تسكينها مسبقاً داخل حاويات التبريد في سيارة السائق (والتي تكون حالتها `IN_FREEZER`)، وذلك تمهيداً لتسليمها في المرحلة النهائية (للمختبر المركزي أو المستشفى).

**المنطق الأساسي للتدفق (بما في ذلك التجميع والحاويات المتعددة):**
1. **التجميع (Grouping):** يختار السائق "بطاقة وجهة" من قائمة المهام. هذه البطاقة تمثل مجموعة مهام مجمعة بواسطة الخادم (`to_location` واحد) وحالتها `IN_FREEZER`.
2. تفتح الكاميرا في البداية لمسح باركود **الحاوية/الثلاجة** الخاصة بالسيارة للتأكد من أنه يفتح الثلاجة الصحيحة.
3. يقوم التطبيق **محلياً** بالتحقق من صحة الحاوية بمطابقتها مع الحاويات المسجلة لسيارة السائق.
4. بعد التأكيد، ينتقل السائق لشاشة مسح الأكياس لسحب الأكياس الموجودة **في هذه الحاوية فقط** والتي تتبع لهذه الوجهة.
5. يبدأ السائق بمسح **أكياس العينات البيولوجية** الفردية لإخراجها. عند مسح كل كيس، يتم إرسال طلب فوراً للخادم لإزالته من الحاوية.
6. **حاويات متعددة:** إذا كان للوجهة أكياس في ثلاجة أخرى، يمكن للسائق الضغط على زر "مسح حاوية" للرجوع لمسح الثلاجة الثانية وسحب أكياسها.
7. بمجرد سحب (مسح وإزالة) جميع الأكياس من القائمة بالكامل لجميع ثلاجات الوجهة، يضغط السائق على زر إنهاء وإغلاق الحاويات لتحديث الحالة الكلية لكل المهام المجمعة دفعة واحدة.

## مخطط التدفق الكامل للمطور

```text
[شاشة اختيار نوع المهمة] 
  ↓ (السائق يضغط على "سحب العينات من الحاوية")
  ↓ يتم تمرير أو فلترة المهام على حالة "IN_FREEZER"
  ↓
[Screen 1: قائمة المهام - InFreezerTaskListFragment]
  ↓ (يضغط السائق على المهمة المطلوبة)
  ↓
[Screen 2: شاشة مسح الحاوية - InFreezerScanFreezerFragment]
  ↓ مسح باركود الحاوية
  ↓ يتم التحقق من الباركود محلياً (Local Validation) 
  ↓ الضغط على زر "PROCEED / متابعة"
  ↓
[Screen 3: شاشة سحب وإزالة الأكياس - InFreezerRemoveBagsFragment]
  ├─► مسح باركود كيس العينة
  ├─► استدعاء API إزالة الكيس (removeBagFromContainer)
  ├─► إزالة الكيس من القائمة المعروضة بنجاح
  └─► تتكرر العملية حتى تفرغ القائمة بالكامل
        ↓
  [عندما تفرغ القائمة (bagList.size() == 0)]
  └── يضغط السائق على "CLOSE FREEZERS / إغلاق الحاويات"
        ↓ استدعاء API الإغلاق (tasks/freezer/out)
        ↓
[Screen 4: شاشة النجاح - TaskStatusFragment]
```

## Screen 1: InFreezerTaskListFragment — شاشة قائمة مهام السحب
### الكلاس الأصلي: `InFreezerTaskListFragment.java`

### 1. التدفق والوصول
- **كيف تصل إليها:** من شاشة اختيار نوع المهمة عند الضغط على زر **SAMPLES PULL OUT / سحب العينات من الحاوية**.
- **إلى أين تأخذك:** عند الضغط على بطاقة أي مهمة، تنتقل إلى شاشة مسح الحاوية (`InFreezerScanFreezerFragment`).

### 2. عناصر التصميم ووصفها الوظيفي
- **شريط التنقل العلوي:** عنوان الشاشة "سحب العينات من الحاوية" (Samples Pull Out)، مع سهم الرجوع.
- **قائمة المهام (RecyclerView):** تعرض المهام التي تحتوي على عينات موجودة حالياً داخل حاويات سيارة السائق، وتنتظر السحب والتسليم.

### 3. المنطق البرمجي والشروط (Business Logic)
```text
عند فتح الشاشة (getTaskList):
├── استدعاء API جلب قائمة المهام (getInFreezerTasks) بحالة "IN_FREEZER".
├── عند نجاح الاستجابة، يتم عرض المهام باستخدام (OutFreezerListTaskAdapter).
└── عند الضغط على أي بطاقة مهمة:
    ├── حفظ المهمة محلياً (UserInfo.getInstance().selectedOutFreezerTask = task)
    └── الانتقال فوراً إلى شاشة (InFreezerScanFreezerFragment)
```

## Screen 2: InFreezerScanFreezerFragment — شاشة مسح باركود الحاوية
### الكلاس الأصلي: `InFreezerScanFreezerFragment.java`

### 1. عناصر الواجهة ووصفها الوظيفي
- **إطار الكاميرا:** لفتح الكاميرا ومسح باركود الحاوية (الثلاجة).
- **نص النتيجة (Scan Result Text):** يعرض رسالة النجاح "Verified" باللون الأزرق أو الفشل "Invalid Container" باللون الأحمر.
- **زر المتابعة (PROCEED / RESCAN):** يكون مخفياً أو يعرض "RESCAN" عند الفشل، ويتغير إلى "PROCEED" عند نجاح المسح.

### 2. المنطق البرمجي (Local Validation Logic)
بخلاف شاشة `SAMPLES PLACEMENT` التي ترسل الكود للخادم للتحقق منه، فإن هذه الشاشة تعتمد على **تحقق محلي كامل**:
```text
عند مسح الباركود:
├── يتم تقسيم الباركود الممسوح باستخدام علامة الناقص "-" (مثال: "12-container")
├── يتم التحقق من أن الجزء الثاني هو كلمة "container"
├── يتم تحويل الجزء الأول إلى رقم (ID الحاوية).
└── البحث داخل حاويات سيارة السائق المحفوظة مسبقاً (UserInfo.loginInfo.getCar().getContainers()):
    ├── إذا تم العثور على تطابق (isExistedContainer == true):
    │   ├── يتم حفظ نوع الحاوية (scannedContainerType)
    │   ├── يتم حفظ رقم الحاوية (scannedContainerId)
    │   ├── إيقاف الكاميرا وعرض رسالة نجاح خضراء/زرقاء.
    │   └── تفعيل زر "PROCEED" للانتقال للشاشة التالية.
    └── إذا لم يتم العثور على تطابق:
        └── عرض رسالة خطأ حمراء، وتغيير الزر إلى "RESCAN" لإعادة المحاولة.
```

## Screen 3: InFreezerRemoveBagsFragment — شاشة سحب وإزالة الأكياس
### الكلاس الأصلي: `InFreezerRemoveBagsFragment.java`

تُعد هذه الشاشة هي الأهم في تدفق السحب. السائق يقوم بمسح الأكياس لإخراجها من الحاوية التي تم التحقق منها للتو.

### 1. المنطق البرمجي للتهيئة وإعداد القائمة
```text
عند فتح الشاشة (getBagList):
├── يتم استخراج قائمة الأكياس (bagIds) من المهمة المحددة مسبقاً.
├── يتم فلترة الأكياس لعرض فقط الأكياس التي تنتمي للحاوية الممسوحة (containerId == scannedContainerId).
└── يتم عرض هذه الأكياس المفلترة في القائمة (RecyclerView) ليقوم السائق بمسحها.
```

### 2. تدفق المسح والإزالة الفردية
هنا، بدلاً من حفظ الأكياس دفعة واحدة (كما في Placement)، يتم استدعاء API لكل كيس بشكل فردي بمجرد مسحه:
```text
عند مسح السائق لباركود كيس:
├── يبحث التطبيق هل الكيس الممسوح موجود ضمن القائمة الحالية؟
│   ├── إذا "نعم": 
│   │   ├── يتم استدعاء API حذف الكيس (removeBagFromContainer).
│   │   └── عند النجاح، يتم حذف الكيس فوراً من القائمة المعروضة وتحديث الواجهة.
│   └── إذا "لا": 
│       └── يظهر تنبيه للسائق بأن الكيس غير صحيح أو غير موجود.
```

### 3. مسار الحاويات المتعددة وتدفق الإغلاق
```text
زر "SCAN FREEZER":
├── يسمح للسائق بالرجوع خطوة للخلف لمسح "ثلاجة/حاوية أخرى" في حال كانت عينات هذه الوجهة موزعة على أكثر من ثلاجة.

زر "CLOSE FREEZERS":
├── عند الضغط عليه، يتحقق التطبيق أولاً (bagList.size() == 0):
│   ├── إذا لم تفرغ القائمة (هناك أكياس لم تُسحب من أي ثلاجة): يظهر خطأ "Please remove all bags to close freezer".
│   └── إذا فرغت القائمة بالكامل: 
│       ├── يظهر رسالة تأكيد للسائق (هل أنت متأكد؟).
│       └── عند الموافقة يتم استدعاء API الإغلاق (closeInFreezer) وتمرير المصفوفة المجمعة `[taskIds]`.
│           └── عند النجاح ← الانتقال إلى شاشة النجاح (TaskStatusFragment).
```

## مدخلات ومخرجات كافة الـ APIs بالتفصيل (API Specifications)

### 1. API: جلب مهام IN_FREEZER
* **الغرض:** جلب المهام المسكنة مسبقاً في الثلاجات وتنتظر السحب.
* **رابط الاتصال (Endpoint):** `POST` لـ `driver/client/tasks`
* **المدخلات (Request Body):**
  ```json
  {
    "driver_id": 23,
    "status": "IN_FREEZER"
  }
  ```
* **المخرجات الناجحة:** قائمة مهام تشمل تفاصيل المواقع و `bagIds` وتفاصيل الحاويات.

### 2. API: إزالة الكيس من الحاوية (removeBagFromContainer)
* **الغرض:** إعلام الخادم بأنه تم سحب الكيس الفلاني من الحاوية، لتحديث حالته وتتبع مكانه.
* **رابط الاتصال (Endpoint):** `POST` لـ `bag/container/remove`
* **المدخلات (Request Body):**
  ```json
  {
    "task_id": 45,
    "bag_code": "BAG-ROOM-001",
    "container_id": 12
  }
  ```
* **المخرجات الناجحة:**
  ```json
  {
    "status": true,
    "message": "Bag removed successfully"
  }
  ```

### 3. API: إغلاق مهام الثلاجات للسحب (closeInFreezer)
* **الغرض:** إنهاء تدفق السحب وتحديث حالة المهام للانتقال للمرحلة النهائية (DELIVERED/OUT_FREEZER).
* **رابط الاتصال (Endpoint):** `POST` لـ `tasks/freezer/out`
* **المدخلات (Request Body):**
  ```json
  {
    "tasks": [45, 46]   // مصفوفة بمعرفات المهام التي اكتمل سحبها
  }
  ```
* **المخرجات الناجحة:**
  ```json
  {
    "status": true,
    "message": "Task freezer closed successfully"
  }
  ```

## ملخص الحالات والمتغيرات المشتركة المطلوبة للمطور (State Management)

المتغيرات المؤقتة التي يعتمد عليها التدفق بالكامل والمخزنة غالباً في `UserInfo`:
```dart
// المتغيرات الخاصة بالمهمة
OutFreezerTaskModel selectedOutFreezerTask;  // المهمة التي اختارها السائق للسحب

// المتغيرات الخاصة بمسح الحاوية
int scannedContainerId;         // رقم الحاوية (ID) المستخرج من الباركود (مثال: 12)
String scannedContainerType;    // نوع الحاوية المستخرج محلياً من بيانات السائق

// المتغيرات المؤقتة داخل شاشة الإزالة
List<BagItemModel> bagList;     // القائمة المفلترة للأكياس المطلوب سحبها من الحاوية الحالية
String tmpScannedBagCode;       // باركود الكيس الذي تم مسحه للتو وينتظر الإزالة
```

> **ملاحظة أمنية بالغة الأهمية:** 
> - يعتمد التطبيق في خطوة مسح الحاوية على الفلترة المحلية والبيانات المخزنة مسبقاً وقت تسجيل الدخول (`loginInfo.getCar().getContainers()`). لذا يجب التأكد دائماً أن بيانات سيارة السائق محدثة، ولا يتم تغيير حاويات السيارة دون إعادة مزامنة (Login/Profile update).
> - عمليات مسح الأكياس هي عمليات لحظية واستدعاءات شبكة فردية (API call per bag). يجب التعامل مع أخطاء الشبكة أثناء إزالة كيس معين بشكل يضمن عدم تعطيل باقي الأكياس.
