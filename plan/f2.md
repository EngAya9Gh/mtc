# تفاصيل شاشات Flutter - الجزء الأول (Part 1: Auth + Home + Medical Tasks)
# المصادقة + الرئيسية + المهام الطبية

> [!NOTE]
> **إجابات على الأسئلة:**
> - **Offline Support**: يعني تخزين البيانات محلياً (مثل قائمة المهام والباركودات) حتى يعمل التطبيق بدون إنترنت ويُرسل البيانات عند عودة الاتصال
> - **Code Generation vs Bloc**: الاثنان ممتازان. Bloc أكثر شيوعاً ومستقراً. Code Generation مع Riverpod يقلل boilerplate code. اختر Bloc إذا الفريق معتاد عليه
> - **إرسال العينات**: لا يوجد تأخير زمني بينها. الإرسال فوري ومتسلسل: Room→ عند النجاح→ Refrigerate→ عند النجاح→ Frozen

## 1. LoginScreen — شاشة تسجيل الدخول ← `LoginActivity`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** هي الشاشة الأولى عند فتح التطبيق (إذا لم يكن المستخدم مسجلاً للدخول)، أو عند الضغط على "تسجيل الخروج" من القائمة الجانبية.
- **إلى أين تأخذك:** إلى `TermsScreen` (إذا لم يوافق على الشروط مسبقاً) أو إلى `MainScreen` (الرئيسية).

### التصميم (UI Layout)
- **الخلفية**: صورة كاملة `bg_login_img` تغطي الشاشة بـ `centerCrop`
- **فوق الخلفية** (طبقة شفافة):
  - عنوان التطبيق "Blazma Logistics" بخط `Nunito Bold` حجم 35، لون أزرق `colorMainTextBlue`، مع margin top كبير (~120dp)
  - **حقل رقم الجوال** (`etUsername`): خلفية مستديرة بظل `rounded_shadow_bg`، ارتفاع 48dp، `inputType=phone`، hint="Mobile"
  - **حقل كلمة المرور** (`etPassword`): نفس التصميم، `inputType=text`، hint="Password"
  - **صف "تذكرني"**: CheckBox (مفعل افتراضياً) + نص "Remember Me" بلون أزرق
  - **زر LOGIN**: عرض كامل، ارتفاع 48dp، لون أزرق `colorMainTextBlue`، نص أبيض، زوايا مستديرة

### اللوجيك والشروط
```
عند فتح الشاشة:
├── إذا KEY_IS_LOGGED_IN == true:
│   ├── إذا KEY_IS_ACCEPT_TERMS == false → انتقل لشاشة الشروط
│   └── إذا KEY_IS_ACCEPT_TERMS == true → انتقل للرئيسية
└── إذا KEY_IS_LOGGED_IN == false:
    └── اعرض نموذج الدخول
    └── إذا KEY_IS_REMEMBER_ME == true → املأ الحقول من التخزين المحلي

عند الضغط على LOGIN:
├── تحقق: username و password غير فارغين (وإلا Toast خطأ)
├── استدعِ API: POST driver/login مع {username, password, fcm_token}
├── عند النجاح (status == true):
│   ├── احفظ username + password في SharedPreferences
│   ├── احفظ حالة remember_me
│   ├── احفظ KEY_IS_LOGGED_IN = true
│   ├── احفظ بيانات المستخدم في UserInfo.loginInfo
│   ├── احفظ user_id
│   ├── طبّق اللغة من السيرفر (language field)
│   ├── إذا termAccepted == true → انتقل للرئيسية
│   └── إذا termAccepted == false → انتقل لشاشة الشروط
└── عند الفشل: اعرض رسالة الخطأ
```

### API المستخدمة
| Endpoint | Method | Body | Response |
|----------|--------|------|----------|
| `driver/login` | POST | `{username, password, fcm_token}` | `LoginResponse{status, message, data: LoginData}` |

## 2. TermsScreen — شاشة الشروط والأحكام ← `TermsConditionActivity`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `LoginScreen` بعد نجاح تسجيل الدخول لأول مرة.
- **إلى أين تأخذك:** إلى `MainScreen` بعد التوقيع والموافقة.

### التصميم (UI Layout)
- **شريط علوي**: زر رجوع (←) + عنوان
- **زرّا اللغة**: "Arabic" و "English" لفتح روابط PDF الشروط في المتصفح
- **منطقة التوقيع**: `SignatureView` مخصص (Canvas للرسم بالإصبع)
- **زر حذف التوقيع**: يمسح Canvas التوقيع
- **زر "Accept"**: لقبول الشروط وإرسال التوقيع

### اللوجيك والشروط
```
عند فتح الشاشة:
├── استدعِ GET driver/terms/get لجلب روابط PDF (عربي + إنجليزي)
├── إذا UserInfo.loginInfo == null:
│   └── أعد تسجيل الدخول تلقائياً من SharedPreferences
└── اعرض أزرار اللغة مع الروابط

عند الضغط على Arabic/English:
└── افتح الرابط في المتصفح الخارجي

عند الضغط على Accept:
├── تحقق: التوقيع غير فارغ (SignatureView ليس فارغاً)
├── حوّل التوقيع إلى Bitmap → احفظه كملف JPG (جودة 70%)
├── استدعِ POST driver/terms/accept مع {driver_id, signature_file}  (multipart)
├── عند النجاح:
│   ├── احفظ KEY_IS_ACCEPT_TERMS = true
│   └── انتقل للشاشة الرئيسية
└── عند الفشل: اعرض رسالة خطأ
```

### API المستخدمة
| Endpoint | Method | Body | Response |
|----------|--------|------|----------|
| `driver/terms/get` | GET | — | `{status, data: {arabicLink, englishLink}}` |
| `driver/terms/accept` | POST (multipart) | `driver_id` + ملف التوقيع | `DefaultResponse` |

### اللوجيك والشروط
```
عند فتح الشاشة:
├── إذا UserInfo.loginInfo == null:
│   ├── أعد تسجيل الدخول من SharedPreferences (POST driver/login)
│   ├── عند النجاح → حدّث بيانات آخر مهمة
│   └── عند الفشل → ارجع لشاشة الدخول (LoginScreen)
├── إذا loginInfo موجود:
│   └── اعرض بيانات آخر مهمة (latestTask)
├── إذا اللغة عربية (ar):
│   └── اعكس اتجاه أسهم الأزرار (rotation = 180)

عند الضغط على MEDICAL TASK (tab_medical_task):
└── انتقل لـ TaskTypeSelectorScreen

عند الضغط على SWAP (tab_swap):
└── انتقل لـ SwapTaskListScreen

عند الضغط على PHARMA TASK (tab_pharma_task):
└── لا يفعل شيء (معطّل حالياً)

عند الضغط على MONEY TASK (tab_car_task):
└── انتقل لـ MoneyTransferListScreen

عند الضغط على Emergency (🔔):
├── اعرض حوار تأكيد "Do you really want to make an emergency request?"
│   (بالعربي: "هل أنت متأكد من إرسال طلب طوارئ؟")
├── عند التأكيد: POST emergency مع {driver_id, car_id}
└── اعرض الرد كـ Toast

عند الضغط على Logout:
└── امسح KEY_IS_LOGGED_IN → ارجع لشاشة الدخول (LoginScreen)
```

### Timeline Logic
```
الحالة NEW     → مؤشر عند index 0 + أيقونة سيارة
الحالة COLLECTED → مؤشر عند index 1
الحالة IN_FREEZER/OUT_FREEZER → مؤشر عند index 2
الحالة CLOSED  → مؤشر عند index 3
```

## 5. TaskListScreen — قائمة المهام ← `ListTaskFragment`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `TaskTypeSelectorScreen` عند اختيار أي من الأزرار الثلاثة.
- **إلى أين تأخذك:** 
  - لمهام `NEW` ← `TaskMapScreen` (الخريطة).
  - لمهام `COLLECTED` ← `FreezerOutBagsScreen` (الفريزر).
  - لمهام `OUT_FREEZER` ← `DeliveryLocationCheckScreen` (التسليم).

### التصميم (UI Layout)
- **شريط علوي**: زر رجوع + عنوان يتغير حسب النوع:

| حالة المهمة | EN Title | AR Title |
|-------------|----------|----------|
| `NEW` | **PICKUP SAMPLES** | **استلام العينات** |
| `COLLECTED` | **SAMPLES PLACEMENT** | **وضع العينات في الحاوية** |
| `IN_FREEZER` | **SAMPLES PULL OUT** | **سحب العينات من الحاوية** |
| `OUT_FREEZER` | **DROP OFF SAMPLES** | **تسليم العينات** |

- **زر "Accept All" / "قبول الكل"**: يظهر فقط إذا يوجد مهام غير مؤكدة
- **SwipeRefreshLayout**: سحب للتحديث
- **RecyclerView**: قائمة المهام، كل عنصر يحتوي:
  - اسم العميل (عربي أو إنجليزي حسب اللغة)
  - **FROM / من** → **TO / إلى** (أسماء المواقع)
  - التاريخ والوقت
  - أيقونة Swap (لطلب تبادل)
  - حالة التأكيد

### اللوجيك والشروط
```
عند فتح الشاشة (onResume):
└── استدعِ POST driver/tasks مع {driver_id, status: selectedTaskType}

عند الضغط على مهمة:
├── احفظ المهمة المختارة في UserInfo.selectedTask
├── إذا selectedTaskType == NEW:
│   └── انتقل لـ ListMapFragment (الخريطة)
├── إذا selectedTaskType == COLLECTED:
│   └── انتقل لـ FreezerOutBagsScanFragment (مسح الأكياس)
├── إذا selectedTaskType == IN_FREEZER:
│   └── انتقل لـ FreezerScanContainerBarcodeFragment (مسح الحاوية)
└── إذا selectedTaskType == OUT_FREEZER:
    └── انتقل لـ ListTaskScanLocationIdFragment (مسح الموقع)

عند الضغط على Accept All:
├── اجمع IDs المهام غير المؤكدة (confirmed_received_by_driver != 1)
├── استدعِ POST driver/tasks/confirm مع {task_ids[]}
└── عند النجاح: أعد تحميل القائمة

عند الضغط على Swap في عنصر:
├── اعرض حوار تأكيد "هل تريد طلب تبادل؟"
├── عند التأكيد: POST swap/create مع {driver_id, task_id}
└── عند النجاح: أعد تحميل القائمة
```

### API المستخدمة
| Endpoint | Method | Body |
|----------|--------|------|
| `driver/tasks` | POST | `{driver_id, status}` |
| `driver/tasks/confirm` | POST | `{task_ids[]}` |
| `swap/create` | POST | `{driver_id, task_id}` |

## 6. TaskMapScreen — شاشة الخريطة ← `ListMapFragment`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `TaskListScreen` عند الضغط على مهمة حالتها `NEW`.
- **إلى أين تأخذك:** إلى `SampleCollectionScreen` (جمع العينات) بعد تأكيد الوصول وبدء المهمة (Start Task).

### التصميم
- **Google Map** كامل الشاشة مع Markers لـ:
  - موقع الاستلام (From Location) - pin أخضر
  - موقع التسليم (To Location) - pin أحمر
  - موقع السائق الحالي - pin أزرق
- **بطاقة سفلية** تعرض تفاصيل المهمة:
  - اسم العميل، من/إلى، التاريخ، الوقت
  - عدد الأكياس `numberOfBags`، عداد RT/REF/FRZ
  - رقم الهاتف (قابل للضغط للاتصال)
  - زر "Start Task" أو "Confirm Location" أو "Navigate"

### اللوجيك والشروط
```
عند فتح الشاشة:
├── اعرض Markers على الخريطة
├── حرّك الكاميرا لتشمل جميع النقاط
├── إذا driverConfirm == null أو "0":
│   └── اعرض زر "Confirm From Location"
├── إذا driverConfirm == "1" و confirmed_received_by_driver == 0:
│   └── اعرض زر "Accept Task"  
└── إذا driverConfirm == "1" و confirmed:
    └── اعرض زر "Start Task"

عند الضغط على Confirm From Location:
├── POST driver/task/fromlocation/confirm مع {task_id, driver_id, from_location, lat, lng}
└── عند النجاح: حدّث الشاشة

عند الضغط على Start Task:
├── POST driver/task/start مع {task_id, driver_id, lat, lng}
└── عند النجاح: انتقل لـ ListTaskSecondSampleInfoFragment (جمع العينات)

عند الضغط على رقم الهاتف:
└── افتح تطبيق الاتصال
```

## 7. SampleCollectionScreen — شاشة جمع العينات ← `ListTaskSecondSampleInfoFragment`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `TaskMapScreen` بعد الضغط على Start Task.
- **إلى أين تأخذك:** 
  - لفتح الماسح ← `BarcodeScanScreen` (أو mobile_scanner مباشر).
  - لإنهاء الجمع ← `SignatureSubmitScreen` (شاشة التوقيع).

### التصميم
- شريط علوي: زر رجوع + عنوان
- **عدّاد الباركودات**: "Total Barcodes: X"
- **Dropdown درجة الحرارة**: Room / Refrigerate / Frozen
- **RecyclerView**: قائمة الباركودات الممسوحة (كود + نوع + حذف)
- **أزرار الإجراءات**:
  - **SCAN BARCODE**: فتح الماسح
  - **SAVE SAMPLES**: إرسال العينات
  - **NO SAMPLES**: لا يوجد عينات
  - **FINISH COLLECTING**: إنهاء الجمع

### اللوجيك والشروط (الأهم في التطبيق)
```
عند الضغط على SCAN BARCODE:
├── احفظ نوع الحرارة المختار من Dropdown
└── انتقل لـ FirstSampleInfoFragment (ثم للماسح)
    *   **ملاحظة عن FirstSampleInfoFragment**: هي شاشة وسيطة لتحديد "بيئة" المسح.
    *   **القائمة المتاحة (Sample Type)**: ثابتة في الكود (Hardcoded) ويجب الحفاظ على ترتيبها:

| الترتيب | API Code (يُرسل للسيرفر) | المسمى بالعربي (UI) |
| :--- | :--- | :--- |
| 0 | `Tubes` | أنابيب |
| 1 | `Swabs` | مسحة |
| 2 | `Body Fluids` | سوائل الجسم |
| 3 | `UBT` | فحص التنفس |
| 4 | `NBS` | كروت الدم |
| 5 | `Blood Bags` | أكياس الدم |
| 6 | `Others` | أخرى |



عند تغيير Dropdown الحرارة:
└── فلتر قائمة الباركودات حسب النوع (ROOM/REFRIGERATE/FROZEN)

عند الضغط على SAVE SAMPLES:
├── تحقق: يوجد باركودات ممسوحة + locationID موجود
├── ابدأ الإرسال المتسلسل (بدون تأخير - فوري عند النجاح):
│   ├── الخطوة 1: فلتر باركودات ROOM → POST samples/add
│   │   ├── Body: {task_id, location_id, barcodes[], container:"ROOM", bag_code, sample_type}
│   │   └── عند النجاح: احذف ROOM من القائمة المحلية → الخطوة 2
│   ├── الخطوة 2: فلتر باركودات REFRIGERATE → POST samples/add
│   │   ├── Body: نفس البنية مع container:"REFRIGERATE"
│   │   └── عند النجاح: احذف REFRIGERATE → الخطوة 3
│   └── الخطوة 3: فلتر باركودات FROZEN → POST samples/add
│       ├── Body: نفس البنية مع container:"FROZEN"
│       └── عند النجاح: Toast "تمت الإضافة بنجاح"
├── ⚠️ إذا كان نوع معين فارغ → تخطاه مباشرة
└── ⚠️ لا يوجد ROLLBACK عند فشل خطوة وسطى!

ملاحظة: كل باركود له خاصية count (العدد)
→ يتم تكرار الكود في المصفوفة بعدد count مرة قبل الإرسال

عند الضغط على NO SAMPLES:
├── حوار تأكيد "هل أنت متأكد؟"
├── POST task/nosamples مع {task_id}
└── عند النجاح: انتقل لـ DriverSignatureFragment

عند الضغط على FINISH COLLECTING:
└── انتقل مباشرة لـ DriverSignatureFragment
```

## 8. SignatureSubmitScreen — شاشة التوقيع وإرسال المهمة ← `DriverSignatureFragment`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `SampleCollectionScreen` عند الضغط على "Finish Collecting" أو "No Samples".
- **إلى أين تأخذك:** بعد الضغط على Submit Task، تظهر شاشة النجاح ثم تعود للمستخدم لقائمة المهام.

### التصميم
- شريط علوي: زر رجوع + عنوان
- **اسم الموقع**: يعرض `scannedLocationName`
- **RecyclerView**: قائمة العينات المسجلة في السيرفر (مقارنة)
- **عدّاد**: "Total Barcodes: X"
- **منطقة التوقيع**: SignatureView (Canvas) - **حالياً معطّل/غير مطلوب**
- **زر حذف التوقيع**
- **زر SUBMIT TASK**

### اللوجيك والشروط (حرج جداً)
```
عند فتح الشاشة:
├── POST samples/list مع {task_id}
├── إذا selectedTaskType == NEW:
│   └── اعرض العينات من السيرفر مع النوع والحرارة
├── إذا selectedTaskType == OUT_FREEZER:
│   └── اعرض العينات الممسوحة محلياً (من UserInfo.scannedBarCodes)
│       واحفظ عينات السيرفر في registeredSampleList للمقارنة

عند الضغط على SUBMIT TASK:
├── إذا selectedTaskType == NEW (مهمة جديدة):
│   ├── إذا taskType == "BOX":
│   │   └── POST task/collect مع {task_id, box_count, sample_count} (بدون توقيع)
│   ├── إذا taskType != "BOX":
│   │   └── POST task/collect مع {task_id} فقط (بدون توقيع)
│   └── عند النجاح: EventBus → ارجع لقائمة Freezer Placement
│
├── إذا selectedTaskType == OUT_FREEZER (تسليم):
│   ├── تحقق أولاً: الباركودات الممسوحة == الباركودات من السيرفر
│   │   (نفس العدد + نفس الأكواد بالضبط)
│   ├── إذا لم تتطابق: Toast "Scanned items does not match"
│   ├── إذا تطابقت:
│   │   └── POST task/close مع {task_id, deliver_confirmationCode}
│   └── عند النجاح: انتقل لـ TaskStatusFragment
```

> [!WARNING]
> **التوقيع الرقمي معطّل حالياً** في الكود (الأسطر محذوفة). يتم إرسال المهمة بدون توقيع.
> كود التأكيد (OTP) الخاص بالتسليم موجود في `DriverTask.otp`

## 9. CarInspectionScreen — شاشة فحص السيارة ← `AddCarImagesActivity`

**التدفق والوصول (Navigation Flow):**
- **كيف تصل إليها:** من `MainScreen` بفتح القائمة الجانبية (Drawer) واختيار **CAR IMAGES**.
- **إلى أين تأخذك:** تعود بك إلى `MainScreen` بعد رفع الصور بنجاح.

### التصميم
- شريط علوي: عنوان "Car Images"
- **6 مربعات صور**: كل مربع يعرض صورة من الكاميرا أو placeholder
- **منطقة التوقيع**: SignatureView
- **زر Upload**: لرفع جميع الصور + التوقيع

### اللوجيك
```
عند فتح الشاشة:
├── اعرض 6 أماكن للصور (أمام، خلف، يمين، يسار، داخل1، داخل2)
└── كل مكان يفتح الكاميرا عند الضغط

عند الضغط على Upload:
├── تحقق: جميع الصور الـ6 ملتقطة + التوقيع موجود
├── POST driver/car/images (multipart):
│   ├── driver_id
│   ├── car_id
│   ├── signature (ملف صورة)
│   ├── image1..image6 (6 ملفات صور)
└── عند النجاح: انتقل للرئيسية
```

### API المستخدمة
| Endpoint | Method | Body |
|----------|--------|------|
| `driver/car/images` | POST (multipart) | `driver_id, car_id, signature, image1-6` |

## ملخص تدفق المهمة الطبية الكامل (Complete Medical Task Flow)

```
Login → Terms (أول مرة) → Main Screen
  → Medical Task → Task Type (NEW/COLLECTED/OUT_FREEZER)
    → Task List → اختر مهمة
      → [إذا NEW]:
          Map → Confirm Location → Start Task
          → Sample Collection (مسح الباركود + اختيار الحرارة)
          → Save Samples (متسلسل: ROOM→REF→FROZEN)
          → Finish Collecting → Signature Screen
          → Submit Task (POST task/collect)
          → ← يرجع لقائمة Freezer Placement
```
